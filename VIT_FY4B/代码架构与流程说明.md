# 一、项目整体架构与设计思路

这个项目的核心流程是：

**原始 FY-4B 卫星场景数据（HDF/NC） → 构建 patch 数据集 → 训练 ViT / CNN → 做数据量消融实验 → 输出结果图、指标表、注意力可视化**

## 3. 核心代码思路

## 3.1 数据预处理脚本的思路

文件：`build_fy4b_vit_scene_split_month_stratified.py`

这个脚本做的是：**从原始 FY4B 场景构造机器学习可直接训练的数据集**。

### 处理流程

#### 1）先匹配场景
- 用时间戳匹配 `.HDF` 和 `.NC`
- 保证每个场景同时有观测数据和标签数据

#### 2）先按 scene 划分 train/test，再切 patch
- 不是先切 patch 再随机分训练测试
- 而是先按整景(scene)划分
- 这样可以避免同一场景的相邻 patch 同时出现在 train 和 test，防止数据泄漏

#### 3）做月份分层
- train/test 都覆盖 `01 / 04 / 07 / 10` 月
- 让训练和测试都包含不同季节月份，减少季节偏置

#### 4）切 patch 时做严格筛选
保留 patch 的条件：
- patch 大小是 `64×64`
- 云相标签纯度 ≥ `0.8`
- 只保留 0~4 这五类
- patch 中不能含无效 FDI 像元

#### 5）训练集使用动态 stride
不同类别 stride 不同：
- 清空类 stride 大
- 混合类 stride 小

本质是在按类别难度/稀缺性控制采样密度，让难类或少类得到更多重叠样本。

#### 6）只用训练集统计均值方差
- 计算 15 通道 mean/std
- 再用于 train/test 归一化
- 避免测试集信息泄漏

#### 7）输出标准化数据
最终写出：
- `.npy` 样本数组
- `.csv` patch 元信息
- `.json` 数据集摘要
- scene manifest

## 3.2 训练主脚本的思路

文件：`fy4b_assignment_pipeline.py`

这个脚本相当于整个项目的**实验调度中心**。

### 代码结构

#### 1）配置中心：`AssignmentConfig`
统一管理：
- 路径
- 模型超参数
- batch size / epoch
- ViT 参数
- 训练参数
- 输出路径

#### 2）数据封装：`DataBundle` / `SplitBundle`
- `DataBundle`：装载数据数组和元数据
- `SplitBundle`：装载 subtrain/val/test 的索引和类权重

#### 3）验证集划分：按 scene 划，不按 patch 划
关键函数：
- `select_val_scenes`
- `build_splits`

特点：
- 训练集内部再拆 val
- 按 scene 级别划分
- 通过多次随机试验 + 惩罚函数，找到类别分布更平衡的 val scene 集合

#### 4）数据增强
- `FY4BNpyDataset`
- `SatelliteTrainAugment`

增强方式很轻量：
- 水平翻转
- 垂直翻转
- 90° 旋转

#### 5）模型部分

##### ViT
自定义实现：
- `PatchEmbedding`
- `MultiHeadAttention`
- `MLP`
- `TransformerBlock`
- `FY4BViT`

结构参数：
- 输入：15 通道，`64×64`
- patch size：8
- embed dim：256
- depth：6
- heads：8
- 分类 token + 位置编码

##### CNN baseline
- 基于 `resnet18`
- 第一层改成 15 通道输入
- 去掉大步长 maxpool
- 最后分类层改成 5 类

#### 6）训练循环
核心函数：
- `run_one_epoch`
- `train_single_experiment`

训练策略包括：
- `AdamW`
- `CosineAnnealingLR`
- 类别加权交叉熵
- label smoothing
- AMP 混合精度
- early stopping
- 以 **val macro-F1** 选最优模型

#### 7）实验设计
`run_full_suite` 自动做：
- 模型对比：ViT / CNN
- 数据量消融：100% / 20% / 10%
- 汇总结果表
- 画速度-精度图
- 生成 attention maps

#### 8）注意力可视化
关键函数：
- `choose_attention_samples`
- `generate_attention_maps`

流程：
- 从测试集里挑每类一个预测正确的样本
- 取 ViT 最后一层 CLS attention
- 插值回原图大小
- 输出原图、attention 热力图、overlay 叠加图

## 4. 整个项目背后的设计思想

### 1）先防数据泄漏
- 先按 scene 划分，再切 patch
- train/val/test 都尽量按 scene 隔离

### 2）遥感数据不是普通图像，采样策略有针对性
通过：
- purity threshold
- invalid pixel 过滤
- 类别动态 stride

在 patch 提取阶段控制数据质量和类别分布。

### 3）模型只是中间环节，实验流程才是主体
项目重点不只是训练一个模型，而是完整实验闭环：
- 数据构建
- 标准化训练
- 基线比较
- 消融实验
- 可解释性输出
- 最终汇总

### 4）用 macro-F1 驱动决策
- 不只看 accuracy
- 更关注类别不平衡时各类的表现

### 5）工程上偏可复现
代码把以下信息全部落盘：
- 配置
- 数据摘要
- split 信息
- 训练历史
- checkpoint
- 图表

从结果文件看，大致可以总结为：

- CNN 最终精度略高于 ViT

# 三、`fy4b_assignment_pipeline.py` 逐函数解释

下面按代码从上到下的顺序说明。

---

## 1. 配置和基础工具

### `AssignmentConfig`
**作用：** 整个实验的统一配置中心。

**管理内容：**
- 数据路径
- 输出路径
- 图像尺寸、通道数、类别数
- ViT 超参数
- batch size、epoch、学习率
- 是否 AMP、是否 GPU

**意义：**
- 避免参数散落在代码里
- 修改实验设置时更集中

---

### `set_seed(seed)`
**作用：** 固定随机种子，尽量保证实验可复现。

**它做了什么：**
- 固定 `random`
- 固定 `numpy`
- 固定 `torch`

---

## 2. 数据封装

### `DataBundle`
**作用：** 把数据和元数据打包在一起。

**包含：**
- `train_x / train_y`
- `test_x / test_y`
- `train_meta / test_meta`
- `train_scene_manifest`

---

### `SplitBundle`
**作用：** 把训练真正需要的 split 信息打包。

**包含：**
- `subtrain_indices`
- `val_indices`
- `test_indices`
- `subtrain_scenes`
- `val_scenes`
- `class_weights`

---

### `load_data_bundle(data_dir)`
**作用：** 从 `data_processed/...` 读取处理好的数据。

**读取内容：**
- `.npy` 样本
- `.csv` 元数据

**特点：**
- 使用 `np.load(..., mmap_mode="r")`
- 通过内存映射降低内存压力

---

## 3. 验证集划分逻辑

### `select_val_scenes(train_meta, num_classes, val_ratio, seed, trials)`
**作用：** 从 train scene 中再挑一部分 scene 当验证集。

**核心思想：**
- 不是随机抽 patch
- 而是随机抽 **scene**
- 试很多次（默认 2000 次）
- 选择类别分布最平衡的一组

**具体步骤：**
1. 统计每个 scene 中各类别 patch 数量
2. 按月份把 scene 分组
3. 每个月抽一定数量 scene 进 val
4. 计算 penalty：
   - val 是否缺某类
   - train 是否缺某类
   - 类别比例是否接近目标
5. 取 penalty 最小的方案

**意义：**
- 验证集更稳
- 类别更均衡
- 还保留月份分层特征

---

### `build_splits(bundle, cfg)`
**作用：** 正式构建 subtrain / val / test 的索引。

**流程：**
1. 用 `select_val_scenes()` 选 val scene
2. 剩余 scene 作为 subtrain
3. 从 metadata 找到对应样本索引
4. 根据 subtrain 标签计算类别权重

**输出：**
- `SplitBundle`

---

### `stratified_subsample_indices(indices, labels, fraction, seed)`
**作用：** 在做 100% / 20% / 10% 数据量消融时，对 subtrain 做按类分层抽样。

**意义：**
- 保证抽样后每类都有样本
- 避免小比例训练时类别崩掉

---

## 4. 数据增强和 Dataset

### `SatelliteTrainAugment`
**作用：** 训练阶段做轻量数据增强。

**增强方式：**
- 左右翻转
- 上下翻转
- 90° 旋转

**为什么合理：**
- 遥感 patch 通常不依赖固定朝向
- 这种增强不会破坏语义

---

### `FY4BNpyDataset`
**作用：** PyTorch 的 Dataset 封装。

**做的事：**
- 根据索引从 `.npy` 中取样本
- 转成 `torch.Tensor`
- 返回 `(x, y)`

---

### `make_loaders(bundle, split_bundle, cfg, fraction)`
**作用：** 构建 train / val / test 三个 DataLoader。

**流程：**
1. 对 subtrain 做分层抽样
2. 创建三个 Dataset
3. 包装成 DataLoader

**输出：**
- `train_loader`
- `val_loader`
- `test_loader`
- 当前 fraction 对应的训练索引

---

## 5. ViT 模型部分

### `PatchEmbedding`
**作用：** 把输入图像切成 patch 并映射到 embedding 空间。

**这里的实现：**
- 使用 `Conv2d(kernel=patch_size, stride=patch_size)` 代替手工切块
- 输入 `[B, 15, 64, 64]`
- 输出 patch token 序列

---

### `MultiHeadAttention`
**作用：** 标准多头自注意力。

**功能：**
- 计算 Q/K/V
- 做 scaled dot-product attention
- 输出 token 间关系
- 可选返回 attention map

---

### `MLP`
**作用：** Transformer block 内部的前馈网络。

---

### `TransformerBlock`
**作用：** 标准 Transformer 编码块。

**结构：**
- `LN -> MHA -> residual`
- `LN -> MLP -> residual`

---

### `FY4BViT`
**作用：** 整个 ViT 分类模型。

**主要组件：**
- `PatchEmbedding`
- `cls_token`
- `pos_embed`
- 多层 `TransformerBlock`
- `LayerNorm`
- 分类头 `Linear`

**关键方法：**
- `forward_features()`：提取 token 特征
- `get_last_selfattention()`：获取最后一层 attention，用于可视化
- `forward()`：输出分类 logits

---

### `build_vit(cfg)`
**作用：** 根据配置创建 ViT。

---

## 6. CNN baseline

### `build_cnn(cfg)`
**作用：** 创建 CNN baseline。

**做法：**
- 基于 `resnet18(weights=None)`
- 第一层改为 15 通道
- 去掉 maxpool
- 最后一层改成 5 类

---

### `build_model(model_name, cfg)`
**作用：** 统一模型工厂。

**逻辑：**
- `"vit"` → `build_vit`
- `"cnn"` → `build_cnn`

---

### `count_parameters(model)`
**作用：** 统计模型参数量。

---

## 7. 训练/评估基础函数

### `evaluate_predictions(targets, preds)`
**作用：** 根据预测结果计算指标。

**输出：**
- accuracy
- macro-F1
- weighted-F1

---

### `run_one_epoch(model, loader, criterion, device, *, optimizer=None, scaler=None, use_amp=False)`
**作用：** 跑一个 epoch；训练和评估都由它统一完成。

**逻辑：**
- 如果传了 optimizer，就是训练模式
- 否则就是评估模式
- 前向、计算 loss、反向、更新参数
- 收集所有预测与标签
- 返回：
  - loss
  - acc
  - macro_f1
  - weighted_f1
  - preds / targets
  - time_sec

这是整个文件最核心的通用执行函数之一。

---

## 8. 可视化与保存

### `save_history_plot(history, output_path)`
**作用：** 把训练历史画成图。

**图中包含：**
- loss
- accuracy
- macro-F1
- 每个 epoch 耗时

---

### `save_confusion_plot(y_true, y_pred, class_names, title, output_path)`
**作用：** 保存混淆矩阵图。

---

### `load_checkpoint_state(path)`
**作用：** 加载 checkpoint，并兼容两种格式：
- 完整 checkpoint dict
- 只有 state_dict 的 dict

---

## 9. 单次实验主函数

### `train_single_experiment(model_name, fraction, bundle, split_bundle, cfg, *, skip_existing=False)`
**作用：** 整个训练流程的核心总函数。

它负责一次完整实验，例如：
- ViT + 100%
- ViT + 20%
- CNN + 10%

---

### 内部流程

#### 第一步：确定输出目录
例如：
- `runs/assignment_suite/vit_fraction_100`
- `runs/assignment_suite/cnn_fraction_020`

#### 第二步：如果结果已存在且允许跳过，就直接复用
- `skip_existing=True` 时不会重复训练

#### 第三步：初始化实验
- 设置随机种子
- 创建模型
- 统计参数量
- 建立 DataLoader

#### 第四步：为当前 fraction 重新计算类别权重
因为 20% / 10% 子样本的类分布可能和全量不同。

#### 第五步：定义训练配置
- loss：带 class weight 的交叉熵
- train loss 含 label smoothing
- optimizer：AdamW
- scheduler：CosineAnnealingLR
- scaler：AMP 混合精度

#### 第六步：epoch 训练循环
每一轮都执行：
1. `run_one_epoch(train)`
2. `run_one_epoch(val)`
3. scheduler 更新
4. 记录 history
5. 如果 val macro-F1 变好，就保存 checkpoint
6. 如果连续很多轮没提升，就 early stop

#### 第七步：训练结束后加载最佳模型
- 用最佳 checkpoint 再跑一遍 val/test
- 确保最终指标来自最佳模型，而不是最后一轮模型

#### 第八步：保存产物
- `history.png`
- `val_confusion_matrix.png`
- `test_confusion_matrix.png`
- `history.json`
- `summary.json`

#### 第九步：返回 summary
供总控函数汇总成最终表格。

## 11. 注意力图相关函数

### `normalize_to_01(img)`
**作用：** 把图像归一化到 `[0,1]`，便于显示。

---

### `build_display_image(sample)`
**作用：** 把 15 通道样本转成可视化图像。

**做法：**
- 取前 3 个通道当伪 RGB
- 若只有 1 通道则做灰度显示

---

### `choose_attention_samples(model, bundle, split_bundle, cfg, *, max_per_class=1)`
**作用：** 从测试集中为每类挑样本做注意力图。

**挑选标准：**
- 每类最多 1 个
- 必须预测正确

**意义：**
- 保证可视化样本具有代表性
- 避免错误样本导致解释混乱

---

### `generate_attention_maps(checkpoint_path, bundle, split_bundle, cfg, *, output_dir=None)`
**作用：** 真正生成注意力图。

**流程：**
1. 加载 ViT 和 checkpoint
2. 用 `choose_attention_samples()` 挑样本
3. 对每个样本：
   - 前向预测
   - 取最后一层 self-attention
   - 提取 CLS 对 patch 的注意力
   - 上采样回 `64×64`
   - 生成三联图：
     - 输入图
     - attention 热力图
     - overlay 叠加图
4. 保存 png
5. 生成 `attention_map_index.json`

# 四、总结

这个文件夹本质上是一个：

**面向 FY-4B 云相分类的完整实验工程**

而不是单纯的“训练某个模型”的脚本集合。

它的主线非常清楚：

1. 原始 FY4B 场景配对  
2. 按 scene 和月份构造 patch 数据集  
3. 训练 ViT 和 CNN  
4. 做数据量消融与时间统计  
5. 输出结果表、对比图、注意力图

如果以后继续扩展，这套结构也很容易增加：
- 新模型
- 新采样策略
- 新评价指标
- 新可解释性方法
